Obsidian Git 플러그인을 사용하여 깃허브와 연동하고, 배포까지 자동화하는 절차입니다. 이 방식은 **백업**과 **출판**을 동시에 해결해 줍니다.

**사전 준비 (터미널 작업)** 플러그인을 켜기 전에, 로컬 폴더를 깃(Git) 저장소로 만들고 깃허브와 연결해야 합니다. 생성해 둔 `my-wiki` 폴더에서 아래 명령어를 한 번만 실행해 주세요.


```
cd my-wiki
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/[아이디]/[저장소명].git
git push -u origin main
```

**플러그인 설치 및 설정** 옵시디언의 '설정 > 커뮤니티 플러그인'에서 **Obsidian Git**을 검색하여 설치하고 활성화합니다. 그 후 플러그인 설정 메뉴로 진입합니다.

**Vault Base Path** 옵시디언 보관소의 최상위 경로가 Git 저장소의 루트(`my-wiki/`)와 일치하는지 확인합니다.

**Auto Backup (자동화 옵션)** 일정 시간마다 자동으로 커밋과 푸시를 하고 싶다면 `Backup interval`을 분 단위(예: 30)로 설정합니다. 수동으로 하고 싶다면 0으로 둡니다.

**제외 파일 설정 (.gitignore)** 불필요한 설정 파일이 공개되지 않도록 루트 폴더에 `.gitignore` 파일을 만들고 반드시 아래 내용을 넣어야 합니다.


```
# [필수] 옵시디언 개인 설정 폴더 제외
.obsidian/

# [필수] MkDocs 빌드 결과물 제외
site/

# [기타] 시스템 파일 및 파이썬 캐시
.DS_Store
__pycache__/
*.py[cod]
```

**사용 방법 (배포 트리거)** 글을 작성한 뒤 깃허브로 보내는 방법입니다.

**수동 배포 (권장)** 명령어 팔레트(`Ctrl + P`)를 열고 `Git: Create backup`을 입력합니다. 이 명령은 `Commit`과 `Push`를 한 번에 수행합니다.

**자동 배포** 위에서 `Backup interval`을 설정했다면, 글을 쓰고 가만히 있어도 설정한 시간 뒤에 자동으로 깃허브로 전송되고 사이트가 갱신됩니다.

**최종 폴더 구조** 옵시디언 보관소(Vault)이자 깃 저장소인 폴더의 모습입니다.

Plaintext

```
my-wiki/
├── .git/                # [시스템] Git 버전 관리 정보가 담긴 숨김 폴더
├── .github/
│   └── workflows/
│       └── publish.yml  # [자동화] 깃허브 액션 배포 스크립트
├── .gitignore           # [설정] 깃허브에 올리지 않을 파일 목록
├── mkdocs.yml           # [설정] MkDocs 사이트 설정 파일
├── requirements.txt     # [설정] 파이썬 패키지 목록
└── docs/                # [데이터] 실제 글이 저장되는 옵시디언 노트 폴더
    ├── index.md         # [페이지] 대문 페이지
    └── assets/          # [리소스] 이미지 및 첨부파일
```

**TIP** 처음 연결 시 인증 오류가 날 수 있습니다. 터미널에서 `git push`를 한 번 실행하여 로그인(자격 증명 저장)을 완료해 두면, 이후 옵시디언 플러그인에서는 별도 로그인 없이 잘 작동합니다.


### github 설정
전 단계에서 만든 로컬 파일들을 받아줄 깃허브 웹사이트 쪽 설정 방법입니다. 순서대로 따라오시면 됩니다.

**1. 저장소(Repository) 생성** 깃허브 사이트 우측 상단의 `+` 버튼을 누르고 `New repository`를 선택합니다.

- **Repository name** 원하는 이름(예: `my-wiki`)을 입력합니다.
    
- **Public / Private** 무료로 사이트를 띄우려면 **Public(공개)**을 선택해야 합니다. (Private은 유료 계정만 Pages 기능을 지원하는 경우가 많습니다.)
    
- **Create repository** 맨 아래 초록색 버튼을 눌러 생성합니다.
    

**2. 액션 권한 부여 (필수)** 깃허브 액션(봇)이 자동으로 배포 브랜치에 글을 쓸 수 있도록 권한을 열어줘야 합니다.

- 생성된 저장소 상단 메뉴의 **Settings** 클릭
    
- 왼쪽 사이드바에서 **Actions** -> **General** 클릭
    
- 스크롤을 맨 아래로 내려 **Workflow permissions** 항목 찾기
    
- **Read and write permissions** 선택
    
- **Save** 버튼 클릭
    

**3. 초기 코드 푸시 (Push)** 이제 로컬(내 컴퓨터)에 있는 파일들을 방금 만든 저장소로 올려야 합니다. 로컬 터미널에서 다음 명령어를 입력합니다. (이미 하셨다면 패스하셔도 됩니다.)

Bash

```
git remote add origin https://github.com/[아이디]/[저장소명].git
git push -u origin main
```

**4. 페이지(Pages) 배포 설정** 코드가 올라가면 `publish.yml`이 작동하여 자동으로 웹페이지용 파일들을 만들어냅니다. 이 파일들이 보일 수 있게 위치를 지정합니다.

- **Settings** 메뉴 다시 클릭
    
- 왼쪽 사이드바의 **Pages** 클릭
    
- **Build and deployment** 섹션의 **Source**를 `Deploy from a branch`로 선택
    
- **Branch** 항목을 `gh-pages` / `/ (root)`로 선택하고 **Save** 클릭 (만약 `gh-pages`가 안 보인다면, 3번 단계 후 1~2분 뒤에 새로고침 해보세요. 액션이 완료되어야 생깁니다.)


옵시디언의 기능을 최대한 살리면서 Material 테마를 적용하기 위한 최적화된 `mkdocs.yml` 설정 파일입니다.

**주요 특징**

- **한국어 설정:** 메뉴와 검색 UI가 한국어로 나옵니다.
    
- **옵시디언 호환성:** `[[링크]]`, 콜아웃(Callout), Mermaid 다이어그램, 체크박스 등을 지원하도록 설정했습니다.
    
- **다크 모드:** 우측 상단 버튼으로 다크/라이트 모드를 전환할 수 있습니다.
    

### `mkdocs.yml` 파일 내용

YAML

```
# 1. 사이트 기본 정보
site_name: 나만의 위키
site_url: https://[깃허브아이디].github.io/[저장소명]
repo_url: https://github.com/[깃허브아이디]/[저장소명]
repo_name: GitHub

# 2. 테마 설정 (Material)
theme:
  name: material
  language: ko             # 한국어 적용
  palette:                 # 다크/라이트 모드 전환 버튼
    - scheme: default
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-7
        name: 다크 모드로 전환
    - scheme: slate
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-4
        name: 라이트 모드로 전환
  features:
    - navigation.tabs      # 상단 탭 네비게이션
    - navigation.top       # '맨 위로' 버튼
    - navigation.indexes   # 폴더 클릭 시 index.md 보여주기
    - content.code.copy    # 코드 복사 버튼

# 3. 플러그인 설정
plugins:
  - search                 # 검색 기능
  - roamlinks              # [[WikiLink]] 문법 지원 (필수)

# 4. 마크다운 확장 기능 (옵시디언 문법 호환)
markdown_extensions:
  - admonition             # 콜아웃 (Callout) 지원
  - pymdownx.details       # 접기/펼치기 (Details) 지원
  - pymdownx.superfences:  # 코드 블록 및 Mermaid 다이어그램 지원
      custom_fences:
        - name: mermaid
          class: mermaid
          format: !!python/name:pymdownx.superfences.fence_code_format
  - pymdownx.tasklist:     # 체크박스 리스트 지원
      custom_checkbox: true
  - pymdownx.highlight:    # 코드 하이라이팅
      anchor_linenums: true
  - pymdownx.inlinehilite
  - attr_list              # 이미지 크기 조절 등 속성 지원
  - tables                 # 표 지원
  - footnotes              # 각주 지원
```

### pip 패키지 설치
```
pip install mkdocs-material mkdocs-roamlinks-plugin
```
### ⚠️ 필수 추가 작업: `requirements.txt` 작성

위 설정 파일에서 `roamlinks` 플러그인을 사용했으므로, 깃허브 액션이 배포할 때 이 플러그인을 설치할 수 있도록 목록을 만들어줘야 합니다. 루트 폴더의 `requirements.txt` 파일에 아래 내용을 저장해 주세요.

Plaintext

```
mkdocs-material
mkdocs-roamlinks-plugin
```
