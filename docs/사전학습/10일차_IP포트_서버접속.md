## 핵심개념

### IP 주소 (공인/사설)

IP 주소는 네트워크에서 각 장치를 식별하는 고유한 주소입니다. 인터넷 세계의 "집 주소"와 같습니다.

**IPv4 주소 형식:**

```
192.168.0.1
 │   │   │ │
 │   │   │ └── 0~255 (호스트)
 │   │   └──── 0~255
 │   └──────── 0~255
 └──────────── 0~255 (네트워크)

총 약 43억 개의 주소 사용 가능
```

**공인 IP vs 사설 IP:**

| 구분 | 공인 IP (Public) | 사설 IP (Private) |
|------|-----------------|------------------|
| 범위 | 전 세계 고유 | 내부 네트워크에서만 고유 |
| 할당 | ISP(인터넷 업체)가 부여 | 공유기/라우터가 부여 |
| 인터넷 접속 | 직접 가능 | NAT 변환 필요 |
| 비용 | 유료 | 무료 |
| 예시 | 203.248.252.2 | 192.168.0.10 |

**사설 IP 대역 (RFC 1918):**

| 클래스 | 범위 | 사용 예 |
|--------|------|---------|
| A | 10.0.0.0 ~ 10.255.255.255 | 대규모 기업 |
| B | 172.16.0.0 ~ 172.31.255.255 | 중규모 조직 |
| C | 192.168.0.0 ~ 192.168.255.255 | 가정/소규모 사무실 |

**NAT (Network Address Translation):**

```
┌─────────────────────────────────────────────────────────┐
│                    내부 네트워크                         │
│  PC1: 192.168.0.10                                      │
│  PC2: 192.168.0.11    ──────>  공유기/라우터  ──────>   │
│  PC3: 192.168.0.12             (NAT 변환)               │
│                                    │                     │
└─────────────────────────────────────│─────────────────────┘
                                     │
                              공인 IP: 203.248.252.2
                                     │
                                     ▼
                                 인터넷
```

**특수 IP 주소:**

| IP 주소 | 용도 |
|---------|------|
| `127.0.0.1` | 루프백 (자기 자신) |
| `localhost` | 127.0.0.1의 별칭 |
| `0.0.0.0` | 모든 네트워크 인터페이스 |
| `255.255.255.255` | 브로드캐스트 |

### 포트 (Port) 번호

포트는 하나의 IP에서 여러 서비스를 구분하기 위한 번호입니다. "아파트 호수"와 같습니다.

**포트 개념:**

```
IP 주소 = 아파트 건물 주소
포트 번호 = 호수

203.248.252.2:80   →  203.248.252.2 아파트 80호 (웹 서버)
203.248.252.2:22   →  203.248.252.2 아파트 22호 (SSH)
203.248.252.2:443  →  203.248.252.2 아파트 443호 (HTTPS)
```

**포트 범위:**

| 범위 | 이름 | 설명 |
|------|------|------|
| 0 ~ 1023 | Well-known Ports | 시스템 예약 (관리자 권한 필요) |
| 1024 ~ 49151 | Registered Ports | 일반 애플리케이션용 |
| 49152 ~ 65535 | Dynamic Ports | 임시 연결용 |

**주요 포트 번호:**

| 포트 | 프로토콜 | 서비스 |
|------|----------|--------|
| 20, 21 | FTP | 파일 전송 |
| 22 | SSH | 보안 원격 접속 |
| 23 | Telnet | 원격 접속 (비암호화) |
| 25 | SMTP | 이메일 발송 |
| 53 | DNS | 도메인 이름 변환 |
| 80 | HTTP | 웹 서버 |
| 443 | HTTPS | 보안 웹 서버 |
| 3306 | MySQL | 데이터베이스 |
| 5432 | PostgreSQL | 데이터베이스 |
| 6379 | Redis | 캐시 서버 |
| 8080 | HTTP-alt | 대체 웹 서버 |

### SSH 원격 접속

SSH(Secure Shell)는 네트워크를 통해 다른 컴퓨터에 안전하게 접속하는 프로토콜입니다.

**SSH의 특징:**
- 암호화된 통신 (비밀번호, 데이터 보호)
- 기본 포트: 22
- 텔넷(Telnet)의 보안 대체
- 키 기반 인증 지원

**SSH 접속 흐름:**

```
┌──────────────┐                    ┌──────────────┐
│   클라이언트   │   ──── SSH ────>  │    서버      │
│  (내 노트북)   │      포트 22       │  (가상머신)   │
│              │                    │              │
│  ssh user@   │   <── 암호화 ───   │  sshd 실행   │
│  192.168.0.5 │       터널         │  (데몬)      │
└──────────────┘                    └──────────────┘
```

**SSH vs Telnet:**

| 구분 | SSH | Telnet |
|------|-----|--------|
| 암호화 | O (암호화됨) | X (평문 전송) |
| 포트 | 22 | 23 |
| 보안 | 높음 | 낮음 (사용 금지 권장) |
| 인증 | 비밀번호/키 | 비밀번호만 |

**SSH 키 인증:**

```
┌─────────────────────────────────────────────────────────┐
│  비밀번호 인증: 매번 비밀번호 입력                        │
│  키 인증: 공개키/개인키 쌍으로 자동 인증                  │
│                                                         │
│  개인키 (클라이언트 보관)  ←→  공개키 (서버에 등록)       │
│  ~/.ssh/id_rsa               ~/.ssh/authorized_keys    │
└─────────────────────────────────────────────────────────┘
```

## 실습활동

### 실습 1: 내 가상머신의 IP 확인하기

**목표:** 가상머신의 IP 주소를 확인하고 네트워크 구성을 이해합니다.

#### Step 1: 가상머신에서 IP 확인

```bash
# Ubuntu/Rocky Linux
ip addr

# 또는
ip a

# 출력 예시
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP>
    inet 192.168.0.105/24 brd 192.168.0.255 scope global enp0s3
         │              │
         │              └── 서브넷 마스크 (24 = 255.255.255.0)
         └── IP 주소
```

```bash
# 간단하게 IP만 확인
hostname -I

# 네트워크 인터페이스 상세 정보
ifconfig    # net-tools 설치 필요
```

#### Step 2: 호스트(Windows)에서 IP 확인

```cmd
# Windows 명령 프롬프트
ipconfig

# 출력에서 확인할 항목
이더넷 어댑터 이더넷:
   IPv4 주소 . . . . . . . . : 192.168.0.100
   서브넷 마스크 . . . . . . : 255.255.255.0
   기본 게이트웨이 . . . . . : 192.168.0.1
```

#### Step 3: 네트워크 구성 이해

```
┌─────────────────────────────────────────────────────────┐
│  홈 네트워크 (192.168.0.x)                              │
│                                                         │
│  ┌─────────────┐                                        │
│  │   공유기     │ ← 게이트웨이: 192.168.0.1             │
│  └──────┬──────┘                                        │
│         │                                               │
│    ┌────┴────┬────────┐                                 │
│    │         │        │                                 │
│ ┌──┴──┐  ┌──┴──┐  ┌──┴──┐                              │
│ │ PC  │  │ VM  │  │Phone│                              │
│ │.100 │  │.105 │  │.110 │                              │
│ └─────┘  └─────┘  └─────┘                              │
└─────────────────────────────────────────────────────────┘
```

### 실습 2: 통신 테스트 (Ping)

**목표:** 네트워크 연결 상태를 확인합니다.

#### Step 1: 로컬 통신 테스트

```bash
# 자기 자신 테스트 (루프백)
ping 127.0.0.1
ping localhost

# 결과 예시
PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data.
64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.031 ms
64 bytes from 127.0.0.1: icmp_seq=2 ttl=64 time=0.029 ms
# Ctrl+C로 중단
```

#### Step 2: 게이트웨이 통신 테스트

```bash
# 공유기(게이트웨이) 테스트
ping 192.168.0.1

# 호스트 PC 테스트 (가상머신에서)
ping 192.168.0.100
```

#### Step 3: 외부 인터넷 테스트

```bash
# DNS 서버 테스트
ping 8.8.8.8        # Google DNS

# 도메인으로 테스트
ping google.com

# 결과가 나오면 인터넷 연결 정상
```

#### Step 4: 포트 연결 테스트

```bash
# telnet으로 포트 테스트 (설치 필요)
telnet 192.168.0.105 22

# nc(netcat)으로 테스트
nc -zv 192.168.0.105 22
nc -zv 192.168.0.105 80

# 결과 예시
Connection to 192.168.0.105 22 port [tcp/ssh] succeeded!
```

### 실습 3: SSH 원격 접속

**목표:** 호스트 PC에서 가상머신으로 SSH 접속합니다.

#### Step 1: 가상머신에서 SSH 서버 확인/설치

```bash
# SSH 서버 상태 확인
sudo systemctl status sshd

# SSH 서버 설치 (Ubuntu)
sudo apt update
sudo apt install openssh-server -y

# SSH 서버 설치 (Rocky Linux)
sudo dnf install openssh-server -y

# SSH 서버 시작 및 자동 시작 설정
sudo systemctl start sshd
sudo systemctl enable sshd

# 상태 재확인
sudo systemctl status sshd
```

#### Step 2: 방화벽 설정

```bash
# Ubuntu (ufw)
sudo ufw allow 22/tcp
sudo ufw status

# Rocky Linux (firewalld)
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
sudo firewall-cmd --list-all
```

#### Step 3: Windows에서 SSH 접속

**방법 1: 명령 프롬프트/PowerShell**

```cmd
ssh username@192.168.0.105

# 예시
ssh ubuntu@192.168.0.105

# 첫 접속 시 fingerprint 확인
Are you sure you want to continue connecting (yes/no)? yes

# 비밀번호 입력
```

**방법 2: PuTTY 사용**

1. [PuTTY 다운로드](https://www.putty.org/)
2. Host Name: 192.168.0.105
3. Port: 22
4. Connection type: SSH
5. Open 클릭

#### Step 4: SSH 접속 후 테스트

```bash
# 접속 성공 후 확인
whoami          # 현재 사용자
hostname        # 호스트 이름
pwd             # 현재 디렉토리
uname -a        # 시스템 정보

# 접속 종료
exit
```

### 실습 4: 포트 개방 및 차단 테스트

**목표:** 방화벽을 통해 특정 포트를 개방하거나 차단합니다.

#### Step 1: 현재 방화벽 상태 확인

```bash
# Ubuntu (ufw)
sudo ufw status verbose

# Rocky Linux (firewalld)
sudo firewall-cmd --list-all
```

#### Step 2: 웹 서버 포트(80) 테스트

```bash
# 간단한 웹 서버 실행 (포트 80)
sudo python3 -m http.server 80

# 다른 터미널에서 또는 호스트에서 접속 테스트
curl http://192.168.0.105
# 또는 브라우저에서 http://192.168.0.105
```

#### Step 3: 포트 80 개방

```bash
# Ubuntu
sudo ufw allow 80/tcp
sudo ufw reload

# Rocky Linux
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```

#### Step 4: 포트 차단 테스트

```bash
# Ubuntu - 포트 80 차단
sudo ufw deny 80/tcp
sudo ufw reload

# 또는 규칙 삭제
sudo ufw delete allow 80/tcp

# Rocky Linux - 포트 80 차단
sudo firewall-cmd --permanent --remove-port=80/tcp
sudo firewall-cmd --reload
```

#### Step 5: 열린 포트 확인

```bash
# 현재 열린 포트 확인
sudo ss -tulnp

# 또는
sudo netstat -tulnp

# 출력 예시
tcp   LISTEN  0  128  0.0.0.0:22    0.0.0.0:*   users:(("sshd",pid=1234))
tcp   LISTEN  0  128  0.0.0.0:80    0.0.0.0:*   users:(("python3",pid=5678))
```

## 네트워크 명령어 정리

| 명령어 | 설명 |
|--------|------|
| `ip addr` / `ifconfig` | IP 주소 확인 |
| `ping <IP>` | 네트워크 연결 테스트 |
| `ssh user@host` | SSH 원격 접속 |
| `ss -tulnp` | 열린 포트 확인 |
| `nc -zv host port` | 포트 연결 테스트 |
| `curl http://host` | HTTP 요청 테스트 |
| `traceroute host` | 네트워크 경로 추적 |
| `nslookup domain` | DNS 조회 |
