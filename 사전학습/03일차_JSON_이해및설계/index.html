    3일차: 데이터의 이름표, JSON 형식 이해 및 설계

    핵심개념

    Key-Value 구조

    Key-Value(키-값) 구조는 데이터를 저장하는 가장 기본적인 방식입니다. "이름표(Key)"와 "내용(Value)"의 쌍으로 데이터를 표현합니다.

    일상생활 예시: | Key (이름표) | Value (값) | |--------------|------------| | 이름 | 홍길동 | | 나이 | 25 | | 직업 | 개발자 | | 이메일 | hong@email.com |

    왜 Key-Value인가? - 데이터에 명확한 이름을 붙여 의미 전달 - 순서에 상관없이 원하는 데이터에 바로 접근 - 사람과 컴퓨터 모두 읽기 쉬운 구조

    {
  "이름": "홍길동",
  "나이": 25,
  "직업": "개발자"
}

    JSON 배열과 객체

    JSON(JavaScript Object Notation)은 데이터를 교환하기 위한 표준 텍스트 형식입니다.

    JSON 객체 (Object)

    중괄호 {}로 감싸며, Key-Value 쌍을 콤마로 구분합니다.

    {
  "name": "아이폰 15",
  "price": 1200000,
  "color": "블랙",
  "inStock": true
}

    규칙: - Key는 반드시 큰따옴표(")로 감싸야 함 - Key와 Value는 콜론(:)으로 구분 - 각 쌍은 콤마(,)로 구분 - 마지막 항목 뒤에는 콤마 없음

    JSON 배열 (Array)

    대괄호 []로 감싸며, 여러 값을 순서대로 저장합니다.

    ["사과", "바나나", "오렌지"]

    [1, 2, 3, 4, 5]

    객체와 배열의 조합

    실제 데이터는 객체와 배열을 중첩하여 복잡한 구조를 표현합니다.

    {
  "학교": "서울고등학교",
  "학년": 2,
  "학생들": [
    {
      "이름": "김철수",
      "점수": 95
    },
    {
      "이름": "이영희",
      "점수": 88
    }
  ]
}

    JSON 데이터 타입

    타입 설명 예시
    문자열 큰따옴표로 감싼 텍스트 "Hello"
    숫자 정수 또는 실수 42, 3.14
    불리언 참/거짓 true, false
    null 값 없음 null
    배열 순서 있는 목록 [1, 2, 3]
    객체 Key-Value 쌍 {"key": "value"}

    파이썬 딕셔너리와의 관계

    파이썬의 딕셔너리(Dictionary)는 JSON 객체와 거의 동일한 구조입니다.

    비교표

    구분 JSON Python Dictionary
    객체 표현 {} {}
    배열/리스트 [] []
    문자열 따옴표 큰따옴표만 (") 큰따옴표/작은따옴표 둘 다 가능
    불리언 true, false True, False
    null 표현 null None

    파이썬에서 딕셔너리 사용

    # 딕셔너리 생성
person = {
    "name": "홍길동",
    "age": 25,
    "skills": ["Python", "JavaScript", "SQL"]
}

# 값 접근
print(person["name"])      # 홍길동
print(person["skills"][0]) # Python

# 값 수정
person["age"] = 26

# 새로운 키 추가
person["email"] = "hong@email.com"

# 키 존재 여부 확인
if "name" in person:
    print("이름이 있습니다")

    JSON과 딕셔너리 변환

    import json

# 딕셔너리 → JSON 문자열 (직렬화)
person = {"name": "홍길동", "age": 25}
json_string = json.dumps(person, ensure_ascii=False)
print(json_string)  # {"name": "홍길동", "age": 25}

# JSON 문자열 → 딕셔너리 (역직렬화)
json_data = '{"name": "김철수", "age": 30}'
person_dict = json.loads(json_data)
print(person_dict["name"])  # 김철수

    JSON 파일 읽기/쓰기

    import json

# JSON 파일 쓰기
data = {
    "title": "My Data",
    "items": [1, 2, 3]
}

with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# JSON 파일 읽기
with open("data.json", "r", encoding="utf-8") as f:
    loaded_data = json.load(f)
    print(loaded_data)

    실습활동

    실습 1: 엑셀의 한 줄을 JSON 코드로 변환해보기

    목표: 표 형식의 데이터를 JSON 구조로 변환하는 방법을 익힙니다.

    실습 단계:

    Step 1: 엑셀 데이터 이해

    아래와 같은 엑셀 데이터가 있다고 가정합니다:

    이름 나이 부서 이메일 재직중
    홍길동 28 개발팀 hong@company.com TRUE
    김영희 32 마케팅 kim@company.com TRUE
    이철수 25 인사팀 lee@company.com FALSE

    Step 2: 한 줄(행)을 JSON 객체로 변환

    엑셀의 첫 번째 행을 JSON으로 변환하면:

    {
  "이름": "홍길동",
  "나이": 28,
  "부서": "개발팀",
  "이메일": "hong@company.com",
  "재직중": true
}

    변환 규칙: - 엑셀 헤더(열 이름) → JSON의 Key - 셀 값 → JSON의 Value - 숫자는 따옴표 없이 - TRUE/FALSE → true/false (소문자)

    Step 3: 전체 데이터를 JSON 배열로 변환

    여러 행을 하나의 JSON으로 표현하면:

    {
  "employees": [
    {
      "이름": "홍길동",
      "나이": 28,
      "부서": "개발팀",
      "이메일": "hong@company.com",
      "재직중": true
    },
    {
      "이름": "김영희",
      "나이": 32,
      "부서": "마케팅",
      "이메일": "kim@company.com",
      "재직중": true
    },
    {
      "이름": "이철수",
      "나이": 25,
      "부서": "인사팀",
      "이메일": "lee@company.com",
      "재직중": false
    }
  ]
}

    Step 3: JSON 유효성 검사

    작성한 JSON이 올바른지 확인합니다.

    온라인 검사 도구: - JSONLint - JSON Formatter

    흔한 오류: - 마지막 항목 뒤에 콤마 붙임 ❌ - 작은따옴표 사용 ❌ (큰따옴표만 허용) - Key에 따옴표 누락 ❌

    Step 4: 파이썬으로 읽어보기

    작성한 JSON 파일을 파이썬으로 읽어 확인합니다.

    import json

# JSON 파일 읽기
with open("my_movies.json", "r", encoding="utf-8") as f:
    data = json.load(f)

# 데이터 출력
print(f"컬렉션 제목: {data['title']}")
print(f"총 영화 수: {len(data['movies'])}개\n")

# 각 영화 정보 출력
for movie in data['movies']:
    print(f"제목: {movie['제목']}")
    print(f"평점: {movie['평점']}")
    print(f"한줄평: {movie['한줄평']}")
    print("-" * 30)
